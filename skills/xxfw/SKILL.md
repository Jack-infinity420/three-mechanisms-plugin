# 信息服务

## 进展报告模板

每完成一个里程碑，按以下格式输出：

| 字段 | 内容 |
|-----|------|
| 项目名称 | [项目名] |
| 报告时间 | [YYYY-MM-DD HH:MM] |
| 当前阶段 | [阶段名称] |
| 整体进度 | [XX%] |

| 任务项 | 状态 | 备注 |
|-------|------|------|
| [任务1] | ⬜ 未开始 / 🔄 进行中 / ✅ 已完成 | [说明] |
| [任务2] | ⬜ 未开始 / 🔄 进行中 / ✅ 已完成 | [说明] |

**下阶段计划**：[下一阶段的重点工作]

**风险与问题**：[当前存在的风险或阻塞问题，无可写"无"]

## 关键汇报节点

以下 5 个节点必须主动输出报告：

1. **任务启动确认** — 收到任务后，确认理解并输出执行计划
2. **方案确定后** — 技术路线选定后，同步方案概要
3. **里程碑完成** — 每个阶段性成果产出后
4. **重大问题/风险** — 发现问题时立即同步，不等下一个节点
5. **任务最终完成** — 输出完整总结，关闭任务

## 信息原则

- **准确性**：数据必须真实，不确定的信息标注"待确认"
- **及时性**：重要进展即时同步，不积压到节点才报
- **完整性**：关键信息（进度、决策、风险）不遗漏
- **清晰性**：结构化表达，重点加粗，一目了然

## 最终总结格式

任务完成时输出以下四项：

1. **已完成事项**：实际产出了什么
2. **关键决策**：执行中做了哪些重要抉择及原因
3. **经验教训**：哪些做法有效，哪些需要改进
4. **遗留事项**：未完成的工作及后续建议

## 理性化对照表

跳过汇报节点的常见借口：

| 借口 | 实际 |
|------|------|
| "还没做完，等做完再报" | 里程碑 ≠ 全部完成。每个节点都要报，用户需要感知进度 |
| "没有重大进展，没什么可报的" | 无进展本身就是信息。沉默 = 不确定 |
| "上次报过了，这次差不多" | 进度是增量。每次报告都要刷新数据 |
| "报告太花时间" | 按模板填充 < 2 分钟。问题拖延的代价远大于此 |
| "风险还不确定，等确定了再说" | 不确定的风险也需要报告，标记"待确认"即可 |

## Red Flags — 立即输出报告

- 超过两个节点未输出进展报告
- 风险项在连续两份报告中出现但未升级
- 最终总结缺少"经验教训"或"遗留事项"
- "还没做完没什么好报的"
- "和上次差不多"
- "等问题确定了再报"

**以上任一信号 = 立即输出当前状态报告，不等下个节点。**

## 常见错误

- 只报进度不报风险，导致问题延后暴露
- 报告堆砌细节，缺少结构化的重点提炼
- 节点已过但未触发汇报，用户对状态缺乏感知
